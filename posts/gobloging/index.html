mengenal "gobloging" dan static site generator menggunakan gohugo

Masih ingat dengan mimpi saya dulu

Bikin aplikasi namanya “goblog” ah!

Mungkin karena waktu itu saya pikir itu nama yang menarik. Saat itu memang bloging sedang booming. Seiring waktu popularitas blog saya rasa semakin menurun. Apalagi jika dibandingkan dengan social media yang menurut saya pribadi tidak terlalu se-social media namanya. Tapi, tidak berarti blog bisa ditinggalkan karena menurut saya blog punya fungsi dan kemampuan yang khusus dan lebih dibanding yang lain. Di sini saya mengajak semua untuk mengenal gobloging, static site generator, dan gohugo.

Apakah gobloging?

Ini juga saya tidak tahu. Jujur, itu hanya istilah yang saya buat. ðŸ¤£ Gobloging berasal dari 2 kata Go dan bloging. Di sini saya akan membagikan cara bloging menggunakan Go. Sebenarnya, Go hanyalahadalah programming language dari Google. Tools yang akan saya gunakan di sini adalah gohugo, sebuah static site generator yang dibuat menggunakan Go.

Apa itu static site generator?

Static site generator adalah alat yang dapat membuat (baca: generate) static website dari source code. Kemampuan utama static site generator adalah mampu menghasilkan static website. Berbeda dengan website dynamic yang umumnya melalui proses database atau proses di server, static website langsung memberikan html ke pengunjung. Kita dapat lihat static website unggul dalam kecepatan karena konten dapat langsung disajikan.

Kenapa gohugo?

Static site generator sebenarnya sudah ada banyak, misalnya: jekyll menggunakan ruby, hexo dan gatsbyjs menggunakan javascript, pelican menggunakan python dan masih banyak lagi. Alasan utama saya memilih gohugo adalah saya tertarik dengan golang, golang mudah digunakan, dan golang memberikan hasil yang bagus di beberapa benchmark dibanding bahasa pemrograman lain. Pada umumnya static site generator memiliki proses yang sama dimana kita membuat template website, lalu kita isi konten umumnya menggunakan markdown, terakhir kita generate versi static. Di sini saya pilih menggunakan gohugo versi master dari github. Instalasi gohugo sangat mudah bahkan bisa langsung digunakan karena hanya berupa single binary application. Tapi saya pribadi jauh lebih memilih menggunakan source code. Untuk mencobanya dapat langsung mengikuti tutorial yang sangat mudah untuk dilakukan.

Pengujian

Saya mencoba membandingkan gohugo dengan aplikasi blog yang umum digunakan. Untuk membantu, di sini saya menggunakan layanan VPS KVM dari Cloudkilat. VPS dari cloudkilat saya rasakan memiliki performa yang baik untuk deploy aplikasi. Kenapa KVM, karena disini saya akan memanfaatkan beberapa tools, yaitu: Docker untuk deployment, Prometheus dan cAdvisor untuk pengukuran, dan Grafana untuk visualisasi. Untuk deployment static website saya menggunakan nginx dan sebagai perbandingan saya menggunakan wordpress. Pengujian saya lakukan beberapa kali dengan menggunakan wrk.

Pengujian dilakukan dengan menggunakan parameter -t4 -c100 -d1m yang berarti wrk akan bekerja dengan 4 thread dan 100 koneksi selama 1 menit. Target dari wrk adalah halaman HTML. ada sedikit kesulitan disini karena sulit mengontrol halaman HTML. Berikut beberapa spesifikasi dan kondisi selama pengujian.

  • Halaman static website berupa artikel sederhana (artikel ini tanpa pengujian dan kesimpulan) yang berukuran 11.96 KB (12,244 bytes)
  • Halaman web wordpress hanya hello word “sisa” install yang berukuran 19.76 KB (20,231 bytes)
  • wrk dijalankan dari server dengan CPU 2 GHz dan memory 2GB
  • web berada di server dengan CPU 2GHz dan memory 4GB
  • Pengujian dijalankan satu per satu
  • Selama pengujian wordpress dan static website tetap berjalan
  • Selama pengujian ada aplikasi background tetap berjalan

Jadi??

Pengujian jelas harus dilakukan dengan lebih baik dan lebih banyak data. Pengujian ini belum bisa menjadi patokan untuk memilih tapi dapat menjadi gambaran kemampuan yang dapat diberikan. Harus diingat halaman wordpress lebih besar, halaman static website bahkan hanya 60.52% dari halaman wordpress. Static website juga memanfaatkan CDN untuk beberapa asset yang besar. Dari beberapa test dapat dilihat static website dapat menangani lebih banyak request. Penggunaan memory static website jauh lebih kecil karena hanya menggunakan web server nginx. Akhirnya yang lebih baik tetap ada di kebutuhan dan pilihan masing - masing. Menurut saya, bloging platform jelas memiliki banyak fitur tetapi static website jauh lebih efisien dalam menggunakan resource.
