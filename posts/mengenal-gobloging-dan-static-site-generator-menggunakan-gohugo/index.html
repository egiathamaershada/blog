mengenal "gobloging" dan static site generator menggunakan gohugo

Masih ingat dengan mimpi saya dulu

Bikin aplikasi namanya “goblog” ah!

Alasannya apa?? saya juga lupa kenapa. Mungkin karena waktu itu saya pikir itu nama yang menarik. Saat itu memang bloging sedang booming. Seiring waktu popularitas blog saya rasa semakin menurun. Apalagi jika dibandingkan dengan social media yang menurut saya pribadi tidak terlalu se-social media namanya. Tapi, tidak berarti blog bisa ditinggalkan karena menurut saya blog punya fungsi dan kemampuan yang khusus dan lebih dibanding yang lain. Di sini saya mengajak semua untuk mengenal gobloging, static site generator, dan gohugo.

Apakah gobloging? ini juga saya tidak tahu. Jujur, itu hanya istilah yang saya buat. ðŸ¤£ Gobloging saya buat berasal dari 2 kata, yaitu Go dan bloging. Di sini saya akan membagikan cara bloging menggunakan Go. Sebenarnya, Go hanyalahadalah programming language dari Google. Tools yang akan saya gunakan di sini adalah gohugo, sebuah static site generator yang dibuat menggunakan Go.

Apa itu static site generator? Static site generator adalah alat yang dapat membuat (baca: generate) static website dari source code. Kemampuan utama static site generator adalah mampu menghasilkan static website. Berbeda dengan website pada umumnya yang dynamic melalui proses database atau proses di server, static website langsung memberikan html ke pengunjung. Kita dapat lihat static website unggul dalam kecepatan. Masalah yang dapat langsung kita lihat adalah, static website jelas tidak dynamic. Di sini static site generator bekerja.

Kenapa gohugo? Static site generator sebenarnya sudah ada banyak, misalnya: jekyll menggunakan ruby, hexo dan gatsbyjs menggunakan javascript, pelican menggunakan python dan masih banyak lagi. Alasan utama saya memilih gohugo adalah saya tertarik dengan golang, golang mudah digunakan, dan golang memberikan hasil yang bagus di beberapa benchmark dibanding bahasa pemrograman lain. Pada umumnya static site generator memiliki proses yang sama dimana kita membuat template website, lalu kita isi konten, terakhir kita generate versi statik.

Di sini saya menggunakan gohugo versi master dari github. Instalasi gohugo sangat mudah bahkan bisa langsung digunakan karena hanya berupa single binary application. Tapi saya pribadi jauh lebih memilih menggunakan source code. Untuk mencobanya dapat langsung mengikuti tutorial yang sangat mudah untuk dilakukan.

Untuk tes, saya akan mencoba membandingkan gohugo dengan aplikasi blog yang umum digunakan. Untuk menbantu, di sini saya menggunakan layanan VPS KVM dari Cloudkilat. Kenapa KVM, karena disini saya akan memanfaatkan Docker untuk deployment dan Prometeheus untuk pengukuran, dan Grafana untuk visualisasi.
